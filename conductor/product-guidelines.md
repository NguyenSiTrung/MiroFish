# Product Guidelines: MiroFish

## Prose Style

- **Developer-facing content** (API docs, code comments, technical guides): Technical, precise, and concise. Use standard software engineering terminology.
- **User-facing content** (UI copy, landing pages, marketing): Accessible, engaging, and imaginative. Emphasize the "predict anything" promise.
- Avoid jargon in user-facing contexts; explain AI/ML concepts in plain language when needed.

## Language Strategy

- **Primary language**: English
- **Secondary language**: Chinese (Simplified)
- English is the default for all documentation, code comments, and commit messages.
- Chinese translations are maintained for README and user-facing documentation.
- Variable names, function names, and code identifiers must always be in English.

## Brand Messaging

- **Tagline**: "A Simple and Universal Swarm Intelligence Engine, Predicting Anything"
- **Core narrative**: MiroFish transforms real-world data into interactive digital worlds where thousands of AI agents simulate collective behavior to predict future outcomes.
- **Key themes**: Swarm intelligence, digital sandbox, emergent behavior, "God's-eye view", zero-risk rehearsal

## Visual Identity

- Logo: MiroFish logo (fish-themed branding)
- Color palette: Follow existing UI theme established in the Vue frontend
- Screenshots and demos are integral to the brand -- always include visual examples when showcasing features

## Content Principles

1. **Show, don't tell**: Use demos, screenshots, and video walkthroughs
2. **Dual-audience awareness**: Always consider both technical users and creative/casual users
3. **Concrete examples**: When explaining prediction capabilities, use real scenarios (public opinion, novel endings, financial events)
4. **Open-source ethos**: Credit dependencies and contributors (e.g., OASIS/CAMEL-AI acknowledgment)
